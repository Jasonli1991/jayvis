# JAYVIS 使用說明

> 這是給 JAYVIS 自己「需要時查」的精簡說明（owner 導向、去除開發細節）。
> 此檔**隨 JAYVIS 一起出貨、重建索引時自動灌進知識庫**（不必手動搬進 vault）——你問到 JAYVIS 的設定／功能／怎麼操作時會自動被檢索到，平時不占對話 context。改了這份內容，下次「重建索引」即同步。

## JAYVIS 是什麼
自架的個人 Telegram AI 搭檔。你不在線時，白名單同事可私訊它、或在群組「@ 它」，它用**搭檔人設**（幽默、有台灣味、會表明「我是你的搭檔」、不冒充你本人），依知識庫（Obsidian 筆記＋GitHub commits＋對話記憶）誠實回答；沒把握就老實說、不編造。全部跑在本機（單一檔 SQLite，免資料庫伺服器）。

## 三種模式
- **你本人（私訊，OWNER_CHAT_ID）**：私人搭檔，可用全部動作工具。
- **同事（白名單內，私訊或群組 @）**：搭檔人設、只依知識庫誠實作答；涉及你的私事又無資料就說沒資料、幫你轉達，絕不編造；時效性問題（同事沒搜尋權）會誠實說查不到。
- **群組**：只在被 @ 時回應，但會記錄群組脈絡；非白名單 @ 會婉拒。

| 功能 | 私訊 | 群組(@) | 同事 |
|---|:--:|:--:|:--:|
| 知識庫問答 | ✅ | ✅ | ✅ |
| 時事搜尋(Tavily) | ✅ | ✅ | ❌ |
| 生圖 | ✅ | ✅ | ❌ |
| 媒體(去背/轉檔/調尺寸) | ✅ | ✅ | ❌ |
| 網站瀏覽 | ✅ | ❌ | ❌ |
| 行事曆 / 收發信 | ✅ | ❌ | ❌ |

## 你（owner）能用的功能，及各自的開啟方式與前置需求
動作工具預設**全關**，在控制台對應卡片開啟，改完多需**重啟 bot**。各工具的「真實可用」條件：

| 功能 | 在哪開 | 前置需求（缺了就做不到） |
|---|---|---|
| 行事曆查/排/改/刪 | 動作工具卡（`ACTIONS_ENABLED`） | **需 macOS**（靠 AppleScript 控制 Calendar.app）；寫入前會先問你確認 |
| 收發信 | 動作工具卡（`EMAIL_ENABLED`）+ 寄件帳號 | **需 macOS Mail 有可用帳號**；寄出前確認 |
| 媒體（去背/轉檔/調尺寸） | 動作工具卡（`MEDIA_ENABLED`） | 去背需 `rembg`；**文件轉檔需 LibreOffice**（面板可一鍵安裝） |
| 時事搜尋 | 動作工具卡（`SEARCH_ENABLED`） | **需 Tavily 金鑰** |
| 自動配圖 | 動作工具卡（`IMAGE_GEN_ENABLED`） | 免金鑰（Pollinations.AI） |
| 網站瀏覽 | 網站瀏覽卡（`BROWSE_ENABLED`） | **首次需下載專用 Chromium**（面板按鈕）；送出/發布前確認 |
| 程式委派 | `.env` 設 `CODE_ROOT`（本機專案母資料夾） | 子資料夾＝一個專案；流程：問答 →「修復計畫」→「執行」改碼開 PR |
| 分析模式 | 面板「分析」卡，或私訊 `分析：<問題>` | 產出自帶圖表的 HTML 報告存 Obsidian Inbox 並開啟，可「接續修改」 |

> 若某工具現在做不到，原因通常是：沒開（去面板開）、或已開但前置沒備好（如非 macOS、沒郵件帳號、Chromium 沒裝、缺 Tavily 金鑰）。

### 你本人的關鍵字/指令（多為私訊限定）
- 任何問題 → 知識庫問答（查不到會誠實說，並可問你要不要記進 Obsidian）。
- `分析：<問題>` → 深度分析報告（私訊限定）。
- 回「`存`」→ 把上一則問答存進 Obsidian `00_Raw/Inbox`（私訊限定）。
- 傳圖/檔＋「去背 / 轉成 pdf / 縮到 1080 寬」→ 媒體工具；接著只打「去背」＝對剛傳那張圖下指令。
- 「幫我畫一隻貓…」→ 自動配圖。
- 「今天台積電股價？／台北天氣？」→ 時事搜尋。
- 「幫我看 example.com／截圖…」→ 網站瀏覽（私訊限定）。
- 程式問題 →「修復計畫」→「執行」→ 程式委派（私訊限定）。
- 「幫我在行事曆新增…／收信摘要」→ 行事曆/收發信（macOS、私訊限定）。

## 記憶（JAYVIS 怎麼記得你）
- 每個人的私訊歷史存成「對話記憶」，回覆時取最近脈絡＋語意回想（recall）。動作做完（行事曆/寄信/媒體）也會記住，下一輪不會忘記自己剛做了什麼。
- 群組對話另存 per-chat 脈絡（被 @ 時參考），不混入私訊私事。
- **長期認識**：私訊累積到一定輪數，背景用模型抽出「JAYVIS 對你的長期認識」。
- **聊天記憶匯入/匯出**（控制台「記憶管理」卡）：把你在別處（ChatGPT/Claude）與 LLM 的長期記憶匯入，讓 JAYVIS 一開始就認識你。限定 JAYVIS `.json` 格式：

```json
{
  "jayvis_memory_version": 1,
  "person": "owner",
  "turns": [
    {"ts": "2026-01-01 10:00:00", "role": "user", "content": "…"},
    {"role": "assistant", "content": "…"}
  ]
}
```
`role` 須為 `user`/`assistant`、`content` 非空字串、`ts` 選填。匯入走完整記憶管線（分塊＋向量＋整併），需先設好「你的 TG id（owner）」；可勾「匯入後重建長期認識」。

## 請假 / 本週重點 / 請假彙整
- **請假狀態**：依面板設定的**日期區間**自動判定；區間內同事會被告知你請假、何時回來。
- **本週重點**：always-on 背景/交接資訊（平常問答也參考）。可按「幫我擬本週重點」用高階模型草擬（從近期對話/commits/近期筆記/一句方向），編修後再存。「近期筆記」需先重建過一次索引才抓得到。
- **請假彙整**：把請假區間內「同事與搭檔的對話」整理成「已處理＋待辦」，面板顯示並發到你 TG；請假結束自動 DM 你一次。

## 知識庫（RAG）
單一檔 `~/.n/kb.sqlite`。檢索＝dense 向量＋FTS5 全文＋RRF 融合＋rerank，低於門檻就傾向誠實說資料不足。灌檔來源：Obsidian vault、GitHub commits（需先 `brew install gh` + `gh auth login`）。用控制台「重建索引」或 `backfill.py` 重建。**新放進 vault 的筆記（含本說明、Notion 匯出摘要）要重建索引後才查得到。**

## 模型路由
依模型名前綴自動分流：`gemini-*`→Google、`claude-*`→Anthropic、`gpt-*`/`o*`→OpenAI、其它→走 `OPENAI_BASE_URL`（本地 Ollama 免金鑰）。面板「模型」卡可設一般/高階模型與金鑰（金鑰一律遮罩、不吐明文）。撞模型額度時，owner 會被指引到面板把模型路由切到本地 Ollama。

## 用量、花費與即時 Log（被問到 token／花費／log 時據此回答）
- **每次 LLM 呼叫的 token 都有記**：log 會印 `📊 LLM <模型>｜token in X / out Y / 共 Z`，在控制台「**即時 Log**」看得到（每行帶時間戳）。
- **目前沒有「累積用量／花費統計」，也沒有換算成 $ 金額**——只有逐次 token 散在即時 Log 裡，要看就一筆筆看；沒有總額儀表板。（單價各家不一、本地 Ollama 免費，所以還沒做金額換算。）
- **JAYVIS 在 TG 對話裡讀不到即時 log**（沒有「讀 log」這個工具）：即時 Log 是給 **owner 在控制台面板**看的。所以 owner 在對話問「我花了多少／即時 log 顯示什麼」時，要**老實說**：逐次 token 請看控制台「即時 Log」、累積花費統計目前沒有——**別假裝自己看得到 log、也別編數字**。
- 出錯時 JAYVIS 會自動「自我診斷」並 DM 給 owner（見下），所以不是完全盲視。

## 出錯自我診斷
發生未預期錯誤時，JAYVIS 會自動用**你設定的 LLM**把錯誤＋近期 log 變成白話「原因／影響／建議」DM 給你，並格式化成可直接轉給作者的回報。全程本機、不對外回報、只給建議不自動改碼。

## 解除安裝（控制台「解除安裝」卡）
回收 JAYVIS 為運作而裝的外部元件。掃描後分三區：**JAYVIS 安裝的**（可安全移除）／**偵測到的相關項目**（來源不明、可能被別程式共用，預設不勾）／**JAYVIS 資料**（知識庫等）。需輸入 `移除JAYVIS` 二次確認、且先停止 bot。只刪「裝前原本沒有、由 JAYVIS 裝上」的，不誤刪你原有或別程式共用的。移除後：本地模型下次用到會自動重下載；Chromium／LibreOffice 不會自動重裝，要用時於對應卡按「安裝」。

## 系統需求（簡）
- 平台：macOS（Apple Silicon 最佳，本地 embedding/rerank 走 MPS）。Intel/Linux 可跑 bot 但本地模型走 CPU 較慢。
- Python 3.11；記憶體建議 16GB；硬碟建議預留 ≥10GB。
- **首次啟動會自動下載約 6.5GB 本地模型**（embedding＋reranker），需穩定網路；之後可離線。

## 資料儲存（`~/.n/`，不進版控）
`kb.sqlite`（知識庫＋對話記憶）、`allowlist.json`（白名單）、`group_conversations.json`（群組脈絡）、`browse_allowlist.json`、`chrome-browse-profile/`（瀏覽登入）、`installed.json`（卸載清單）。

## 常見問題
- **群組 @ 沒反應？** 多半是 Telegram 群組隱私模式沒關：BotFather `/setprivacy` → Disable →（重要）把 bot 移出再重加群組，或設成群組管理員。
- **改了設定沒生效？** 多數設定 bot 啟動時讀入，改完按面板「重啟」；白名單即時生效、分析模式免重啟。
- **同事問非公事會被擋嗎？** 不會；只有短時間連發一堆閒聊才會踩冷卻閘鎖 60 分；非白名單一律婉拒。
